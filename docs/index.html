Win32 apps, every tenant

Upload an .intunewin template once; the bridge runs the full Intune upload state machine in each tenant and fans out updates to everyone who has the app.

Hire & offboard by contract

Contract-driven provisioning: usage location, licenses, groups, UPN convention. New hires and terminations become one reviewed action, not a checklist.

Mailbox fixes that stick

The "mailbox full / not archiving" repair replaces a ~10-cmdlet Exchange Online dance: archive + auto-expand, retention policy, hidden blockers, assistant nudge.

Known-fix workflows

MFA reset, license repair, password reset, compromised-account lockdown - each one diagnoses transparently, fixes idempotently, then re-diagnoses to prove it worked.

Diagnose → Fix → Verify

Every workflow follows the same honest loop, and every run is recorded - who ran what, against which tenant, with which findings. Ticket notes write themselves.

1 - DIAGNOSE
Read the real state from Graph or Exchange Online and show every finding verbatim: Ok Info Warning Blocker
2 - FIX
Apply the known remediation, step by step, recording each step's success or failure. Steps are idempotent - re-running is always safe.
3 - VERIFY
Re-diagnose immediately and show the post-fix state, so "it should be fixed now" becomes "here is the state that proves it".

Why it's different

  • Non-invasive. No agents, no scripts dropped in customer tenants. The bridge speaks the same Graph and Partner Center APIs Microsoft's own portals use, under the Secure Application Model and per-customer GDAP consent.
  • Transparent. Diagnoses are shown raw. Remediation steps are listed one by one with their outcome. Nothing happens behind the operator's back.
  • Auditable. Workflow runs persist to Postgres with operator identity and full findings; failures can push to a Teams webhook.
  • Self-hosted. Your credentials never leave your infrastructure. Runs on docker-compose for evaluation, Kubernetes (Kustomize + Flux + SOPS) for production.

Try it in two minutes

git clone https://github.com/Spillers-Technology/PartnerCenterBridge.git
cd PartnerCenterBridge
docker compose up --build
# SPA:  http://localhost:8081
# API:  http://localhost:5080  (Swagger at /swagger)

Auth is disabled in compose so you can click through the UI without an identity provider. Acting in real customer tenants additionally needs a partner Entra app and a one-time SAM bootstrap.
